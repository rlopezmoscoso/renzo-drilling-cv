# PROJECT_SPEC.md

## 1. Project Name

**Renzo Lopez Moscoso — Professional CV Website**

## 2. Project Purpose

Create a professional static website that functions as a digital CV and technical profile for Renzo Lopez Moscoso, Senior Drilling Engineer and Drilling Supervisor.

The project is also intended as a practical exercise to learn the workflow between ChatGPT, Codex, Git, GitHub, and static web deployment.

## 3. Target Audience

- Oil & Gas recruiters
- Drilling Managers
- Wells Managers
- Operating companies
- Oilfield service companies
- Engineering consultancies
- International energy-sector employers

## 4. Language

The initial version shall be written in English.

The code structure should allow a future Spanish version, but multilingual functionality is outside the current scope.

## 5. Technical Scope

The website shall be developed using:

- HTML5
- CSS3
- Vanilla JavaScript
- Git
- GitHub
- GitHub Pages or an equivalent static hosting service

The project shall not use:

- React
- Next.js
- Vue
- Angular
- Bootstrap
- Tailwind CSS
- npm packages
- Databases
- Backend services
- CMS platforms
- Authentication
- Server-side contact forms

## 6. Website Structure

The website shall contain two pages:

1. `index.html` — Home
2. `experience.html` — Professional Experience

Shared resources:

- `css/styles.css`
- `js/main.js`
- `assets/images/`
- `assets/icons/`
- `assets/documents/`

## 9. Visual Design Requirements

The visual concept shall communicate:

- Senior engineering experience
- Operational discipline
- Technical precision
- Oil & Gas industry context
- Modern digital engineering

### 9.1 Visual Style

- Executive Technical light visual system
- White and light-gray surfaces
- Navy headings and executive sections
- Petroleum-blue navigation, links, and technical accents
- Copper metric, period, and current-role accents
- Clean alignment and controlled whitespace
- Restrained shadows with fewer elevated cards
- No dark full-page background
- No generic recruitment-template appearance

### 9.2 Suggested Visual Elements

- Executive metrics panel with independent operational facts
- Clean timeline components
- Editorial professional-impact layouts
- Compact rectangular tags
- Thin executive separators and borders
- No well-path graphic, drilling schematic, pressure graph, rig-capacity diagram, or decorative technical plot

### 9.3 Responsive Design

The website shall be mobile-first and tested at minimum for:

- 360 px mobile width
- 768 px tablet width
- 1024 px laptop width
- 1440 px desktop width

## 10. Accessibility Requirements

- Semantic HTML
- Logical heading hierarchy
- Visible keyboard focus
- Sufficient color contrast
- Descriptive link labels
- Alt text for meaningful images
- Navigation usable without a mouse
- Respect `prefers-reduced-motion`
- Avoid text embedded inside images

## 11. Performance Requirements

- No unnecessary dependencies
- Optimized image formats
- Minimal JavaScript
- No console errors
- No broken internal links
- Fast loading on normal mobile connections

## 12. Content Restrictions

The website shall not:

- Invent companies, roles, dates, projects, or metrics
- Publish confidential operational details
- Publish personal-reference telephone numbers
- Publish restricted contractual information
- Use client or operator logos without authorization
- Display generic stock images without approval
- Exaggerate qualifications or responsibilities

The first release is accepted when:

- Both pages load without errors
- Navigation between both pages works
- The mobile menu works
- The layout is responsive
- The content accurately represents the approved professional profile
- There are no broken links
- There are no JavaScript console errors
- The HTML uses semantic structure
- Basic accessibility requirements are met
- The repository contains clear documentation
- The website can be deployed as a static site
